# StateMotion v0.1 Alpha — Installation (Development / Alpha)

This is a **development/alpha** installation process, NOT a final consumer
installer. Paths below use `<REPO>` and `<PREMIERE>` placeholders — substitute
your own local locations; do not hardcode personal paths into committed docs.

## Prerequisites
- Adobe Premiere Pro (version TBD — host verification pending).
- UXP Developer Tools (Adobe CC companion) for loading the development panel.
- Node.js 18+ (panel build only).
- A compiled `StateMotion.aex` (built from the native project; see native docs).

## Native Effect (`StateMotion.aex`)

1. Build the native project for your target (Win/macOS). Output: `StateMotion.aex`.
2. Place exactly **one** copy in the Premiere plugins folder:
   - Windows: `<PREMIERE>/Plug-ins/<locale>/`
   - macOS: `<PREMIERE>/Contents/Plug-ins/`
3. **Verify only one AEX exists.** Duplicate effects cause ambiguous match-name
   resolution and are rejected by the panel's contract checks.
   - Windows: `dir "<PREMIERE>/Plug-ins" /s /b | find "StateMotion"`
   - macOS: `find "<PREMIERE>/Contents/Plug-ins" -name "StateMotion.aex"`
4. Effect match name (permanent): `AE.io.github.anmol2k5.statemotion.effect`
5. Verify version/hash where relevant (record the build SHA in your handoff
   notes). Do not trust an unlabeled `.aex`.

## UXP Panel

1. From the panel package directory:
   ```
   npm ci
   npm run build
   ```
   Output: `dist/main.js` + `dist/main.css`.
2. Load via UXP Developer Tools:
   - Add a "Development" panel pointing at `dist/main.js`.
   - Or package for side-loading (final installer out of scope).
3. Verify the panel loads and shows the **About** tab with product version
   `0.1.0-alpha.1`.

## Known Unsupported / Unverified Areas
All host-dependent behavior is **NOT YET OPERATOR VERIFIED** unless explicitly
confirmed in `v0.1-alpha-host-status.md`. Do not assume any host gate passes.
