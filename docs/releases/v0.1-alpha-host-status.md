## Native render BLOCKER — root cause + fix — 2026-07-19
**Root cause (confirmed via Adobe SDK headers + code trace, systematic-debugging):**
`GlobalSetup` declares no depth capability (`out_flags = 0`), so Premiere's
legacy `PF_Cmd_RENDER` path hands the effect an **8-bit `PF_Pixel8`** world
(4 bytes/pixel; component order `alpha, red, green, blue`; rows padded to
`rowbytes`). The Render adapters `smWorldToPixels`/`smPixelsToWorld`
unconditionally `reinterpret_cast` the world buffer to **`PF_PixelFloat*`**
(16 bytes/pixel). Iterating `row[x]` with a 16-byte stride across a 4-byte
buffer over-reads ~4x past every row and past the buffer end → access violation
→ host reports `A low-level exception occurred in: StateMotion (AEVideoFilter:11)`.
Per `AE_Effect.h`, legacy render worlds are only 8-bit (`out_flags=0`) or 16-bit
(`PF_OutFlag_DEEP_COLOR_AWARE` + `PF_WORLD_IS_DEEP`); float worlds arrive only on
the SmartFX path this effect does not implement — so the float cast was always
wrong on this path. The renderer's own sampler is bounds-guarded and was not at
fault.

**Minimal fix (no architecture change):**
- New pure, SDK-free adapter `statemotion_world_pixels.hpp`
  (`worldToPixels`/`pixelsToWorld`) converts by declared depth — 8-bit (0..255),
  16-bit (0..`PF_MAX_CHAN16`=32768), float pass-through — respects `rowbytes`,
  and normalizes to/from the renderer's [0,1] premultiplied `Pixel`.
- `Render()` selects depth via `PF_WORLD_IS_DEEP` (else 8-bit) and delegates.
- Both identity fast paths now use `smIdentityCopy`, a row-wise copy respecting
  per-world `rowbytes` and the overlapping region, replacing the unsafe
  `output->rowbytes * H` `memcpy` from `src->data` (latent over-read on
  src/dst stride or dimension mismatch).

**Regression test:** `statemotion_world_pixels_test.cpp` — 8-bit padded-stride
read, 8/16-bit normalized ranges, float pass-through, padded round-trip
(padding untouched, every channel byte preserved), `bytesPerPixel` per depth,
and write clamping. **12/12 SDK-free C++ suites PASS** (was 11; +world_pixels).

**AEX rebuild:** PASS — `statemotion_effect.cpp` compiles clean with the fix
under MSVC v142 + Adobe SDK (only benign SDK `C5033` + `MSB8012`).
New `StateMotion.aex`: 31232 bytes, SHA-256
